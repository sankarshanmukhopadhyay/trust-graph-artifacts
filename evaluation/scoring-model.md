# Artifact Evaluation Model

Each candidate essay is scored before artifact creation.

---

## Dimensions

| Dimension | Description | Score |
|----------|-------------|------:|
| Structural Clarity | Can the concept be modeled? | 0–5 |
| Enforcement Potential | Can it affect runtime behavior? | 0–5 |
| Delegation Impact | Does it affect authority transfer? | 0–5 |
| Revocation Semantics | Can control be withdrawn or corrected? | 0–5 |
| Redressability | Can disputes be handled structurally? | 0–5 |
| Signal Strength | Based on engagement tier and related evidence | 0–5 |

---

## Threshold

- **18 or higher** → promote to artifact candidate
- **12–17** → refine and revisit
- **Below 12** → retain as narrative only for now

---

## Output

Each evaluation should produce:

- a structured assessment
- a recommendation
- any open questions that block formalization

---

## Principle

Evaluation prevents:

- premature formalization
- conceptual noise
- governance dilution
