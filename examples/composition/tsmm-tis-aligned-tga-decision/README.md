# TSMM/TIS-Aligned TGA Decision Composition

This example shows how a Trust Graph-derived package can be interpreted as a TSMM decision and projected toward a TIS-compatible executable artifact while preserving TGA provenance.

## Files

1. `01-tga-source-package.json` — source package context and Trust Graph provenance.
2. `02-tsmm-decision-receipt.json` — TSMM-profiled runtime decision receipt.
3. `03-tis-decision-receipt.json` — TIS-compatible executable receipt projection.
4. `04-tga-provenance-extension.json` — TGA-specific interpretation and essay provenance overlay.

## Interpretation

The example separates four concerns that are often collapsed:

- source essays explain governance pressure;
- TSMM models the decision semantics;
- TIS-compatible artifacts support external validation and audit;
- TGA provenance preserves the interpretive lineage of the package.
