# v0.2.0 — TSMM v0.20 Alignment and Executable Governance Profile Hardening

## Summary

This release harmonizes Trust Graph Artifacts with TSMM v0.20.0 and clarifies the repository's role in the executable governance stack.

TGA remains the Trust Graph interpretation and package corpus layer. TSMM remains the semantic model. Where executable artifact contracts are required, TGA packages can now be positioned through the TSMM/TIS alignment model rather than maintaining competing governance semantics.

## Added

- Formal TGA ↔ TSMM binding metadata.
- TSMM v0.20 alignment documentation.
- TGA / TSMM / TIS layering crosswalk.
- TSMM decision receipt profile documentation.
- Outcome vocabulary mapping.
- Assurance posture vocabulary mapping.
- TSMM/TIS-aligned TGA decision composition example.
- Explicit TSMM-profiled decision receipt example and schema.

## Changed

- Updated repository version to `0.2.0`.
- Updated stale receipt and decision-provenance documentation metadata.
- Updated active registry versions to `0.2.0`.
- Updated active artifact schema version markers to `0.2.0`.
- Updated decision receipt schema `$id` to a release-pinned repository URL.

- Added `.gitignore` rules for local operating-system files.
