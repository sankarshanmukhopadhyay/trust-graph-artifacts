# v0.3.0 — Authority Envelopes, Machine Commitments, and High-Risk Infrastructure Governance

## Overview

This release expands Trust Graph Artifacts into a stronger executable governance corpus for agentic authority, machine-generated commitments, verification infrastructure, inference substitution, and high-risk digital infrastructure.

The release converts five Trust Graph source essays into TSMM-native package families:

- The Authority Gap
- When Machines Make Commitments
- Fraud as an Externality of Missing Infrastructure
- Before the Graph
- Governance for High-Risk Digital Infrastructure

## Added package families

### Runtime authority envelope

Adds `patterns/runtime-authority-envelope`, a runtime admissibility wrapper that binds technical actor, accountable controller, principal, mandate, scope, policy version, revocation state, decision receipt, and redress route before consequential agent action is finalized.

### Machine commitment governance

Adds:

- `profiles/machine-commitment-governance-profile`
- `evidence/proof-carrying-commitment-receipt`
- `overlays/commitment-lifecycle-mediation`

These packages model machine commitments as institutional obligations with proof, lifecycle state, reconciliation, and contestability.

### Fraud externality verification

Adds `profiles/fraud-externality-verification-profile`, which treats fraud as a cost externality of missing verification infrastructure, weak issuer accountability, weak registry governance, and governance capture.

### Inference-to-verification controls

Adds:

- `patterns/inference-substitution-gap`
- `patterns/legibility-trap-detector`

These packages distinguish inferred, asserted, verified, and authoritative inputs and flag systems that respond to inference failure with more proxy extraction rather than verification infrastructure.

### High-risk digital infrastructure governance

Adds:

- `profiles/high-risk-digital-infrastructure-governance-profile`
- `overlays/constitutional-digital-infrastructure-controls`
- `evidence/high-risk-governance-assurance-record`

These packages model traceability, reversibility, delegation boundaries, guardrail logic, contestability, public transparency, independent auditability, and non-derogable rights controls.
