# Fraud externalities

## Purpose

The fraud externality profile treats fraud as an infrastructure cost created by unverifiable identity, unverifiable claims, forged provenance, unbounded authority, weak issuer accountability, weak registries, and governance capture.

The repository does not model fraud only as detection after harm. It models the missing verification infrastructure that lets fraud costs move from system designers and issuers to relying parties, victims, and public institutions.

## Control layers

### Issuance layer

The issuer must be authorized, accountable, removable, and auditable.

### Registry layer

The registry must publish admission, suspension, removal, appeal, and anti-capture controls.

### Governance layer

The governance system must define who can challenge claims, how externalized costs are detected, and how remediation is triggered.

## Assurance posture

A verification system that only proves a credential format is not enough. Assurance must cover issuer legitimacy, registry governance, provenance, authority, dispute handling, and the institutional allocation of fraud risk.
