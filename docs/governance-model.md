# Governance Model

The repository models governance as executable authority. Authority must be declared, scoped, explicitly delegated, revocable, and evidenced. Enforcement must bind policy evaluation to an actual system action. Licensing authorities must prove their own legitimacy through governance rules, auditability, and redress mechanisms.


## v0.3.0 governance model extension

The governance model now explicitly distinguishes five surfaces that must not be collapsed into one another:

1. **Identity** — which actor or component is present.
2. **Authority** — what the actor is permitted to make true.
3. **Commitment** — which future obligation, reliance condition, or institutional expectation has been created.
4. **Verification** — which claims, issuers, registries, and provenance trails can be independently checked.
5. **Constitutional boundary** — which effects are prohibited, human-led, reversible, contestable, or subject to public oversight regardless of ordinary policy permission.

This distinction keeps executable governance from becoming ambient authorization. A valid package must show where authority comes from, how it is delegated, how it can be revoked, what evidence proves it was checked, and who can challenge the result.
